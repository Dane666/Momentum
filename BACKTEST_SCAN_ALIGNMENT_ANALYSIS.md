**关键洞察：**

> Backtest应该是Scan逻辑的历史模拟，而不是反过来。

| 因素 | Backtest | Scan | 状态 |
|------|---------|------|------|

| **涨幅范围** | 无限制 | 4.0%-9.2% | ❌ 不一致 |
| **量比过滤** | 无 | >= 1.2（已移除严格过滤） | 🔄 部分对齐 |

| **RSI筛选** | > 85删除 | < 80过滤 | ❌ 不一致 |
| **成交额下限** | 无 | >= 1000万 | ❌ 不一致 |

- 优势：完全一致，可直接应用
- 劣势：历史回测收益下降，策略有效性降低

### 选项B：让Scan同步Backtest的宽松筛选（建议）

- ✅ 移除Scan的市场防御0.22 - **已完成**  
- ⚠️ 但Scan仍有涨幅、RSI、成交额限制
- 这些限制是Scan自身的风险管理需要，不是导致不一致的原因

### 选项C：完全对齐所有参数
需要调整的项：

#### 3.1 涨幅范围对齐
```

```

#### 3.2 RSI门槛对齐

```
当前 Scan: RSI < 80 过滤
建议统一: RSI 80-85 为黄色区间（都过滤）

```
